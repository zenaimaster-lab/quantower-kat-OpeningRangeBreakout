## 2. Design Decisions & Rationale

- **PowerShell ASCII Normalization**: Emojis and smart em-dashes inside `.ps1` files run into encoding issues with older PowerShell engines (v5.1 default in Windows 10/11) when parsed without a Byte Order Mark (BOM). Standardizing output logs to pure ASCII tags (e.g. `[Init]`, `[Diary]`, `[Success]`) solves this permanently and elegantly.
- **Direction-Aware Range Boundaries**: Instead of treating ranges as flat barriers (absolute distance), treating them as directional thresholds matches the underlying trading psychology: once broken, resistance turns to support.

## 3. Constraints & Gotchas

- **Double Python Environments**: Changes to graphify parsers must be updated in both the system Python site-packages directory (`AppData\Local\Programs\Python`) and the `uv` tool isolation directory (`AppData\Roaming\uv\tools`) so that both standard commands and subagent invocations inherit MQL5 C++ support.
